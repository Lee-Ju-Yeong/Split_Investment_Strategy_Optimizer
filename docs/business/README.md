# Business Docs Guide

> Purpose: 이 디렉터리는 공개 레포에서도 안전하게 유지할 수 있는 사업 관련 문서를 둡니다.
> Rule: 고객명, 가격 협상안, 법률 의견서, 계약 초안, 세일즈 파이프라인 같은 민감 정보는 공개 문서에 넣지 않습니다.

## What Can Stay Public
- 제품 포지셔닝
- 공개 가능한 사업모델 검토 메모
- 공개 가능한 오퍼 구조
- 공개 가능한 운영 원칙
- 공개 가능한 출시 경계 문서

## What Must Stay Private
- 고객 실명 또는 잠재 고객 목록
- 세일즈 파이프라인 상태
- 내부 가격표와 할인 정책
- 계약서 초안, SOW 초안, 법률 의견서
- 규제 검토 메모 원문
- 데모 계정, 접근 정보, 토큰, 비밀값

## Private Paths
- `docs/business/private/`
- `docs/legal/private/`
- `docs/business/*.local.md`
- `docs/legal/*.local.md`

## Writing Rule
- 공개 문서에는 `왜`, `현재 결론`, `다음 액션`만 남깁니다.
- 민감한 상업 정보가 필요하면 공개 문서에는 요약만 두고, 상세 내용은 private 경로로 분리합니다.
- 고객 사례가 필요하면 실명 대신 익명화한 역할 기준(`boutique AM`, `RIA`, `family office`)으로 씁니다.
